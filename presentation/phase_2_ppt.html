The Problem

Why Existing
IDS Fail

Most ML-based intrusion detection systems achieve high benchmark accuracy — but collapse in production against novel, unseen attacks.

Core issue: Random train/test splits leak future attack knowledge into training. When these models face true zero-day threats, recall drops to near zero.

Legacy Datasets

KDDCup99 and NSL-KDD lack modern attack vectors and don't represent real-world network topologies

Closed-World Evaluation

Models trained on all attack types score well on benchmarks but can't generalize to novel threats

Our Solution

Train only on benign data → any deviation signals an intrusion, including attacks never seen before

Failure Analysis

Why 98.3% of Attacks Were Missed

At P99 threshold: 1,144,086 attacks missed. Five distinct root causes — each requiring a different architectural fix.

1. Stateless Analysis

Each flow scored independently. SSH brute force only reveals itself across thousands of attempts—invisible flow-by-flow. Fix: sliding window aggregation.

2. Protocol Camouflage

HTTP-based attacks (HOIC, LOIC-HTTP, Bot) use normal HTTP protocol. Flow stats look like browser traffic. Error 0.13–0.43 vs benign 0.08.

3. Conservative Threshold

P99 keeps FPR ≤ 1% (Anti-Alert Fatigue). At P90: recall = 70.5% · F1 = 0.749. The model ranks attacks correctly—threshold is the gating factor.

4. Adversarial Design

Bot malware error = 0.127 ≈ benign 0.081. Built specifically to mimic normal traffic. 7.5% of attacks score below the benign mean entirely.

5. Volume vs. Flow

DDoS is dangerous due to aggregate volume. Our model sees one packet at a time, not the massive wave. Fix: Network-level flow frequency sensors.

Key insight: The model does score most attacks higher than benign (4.3× separation) — but 98.3% still fall below the P99 line. Choosing P90 would catch 70.5% of attacks. The right threshold depends on the SOC's analyst capacity.

System Design

Multi-Tier
Defense Stack

No single model handles all threat types. Each tier targets a different weakness in the detection pipeline.

Tier 1 — Random Forest

Supervised classifier trained on known attack labels. Perfect precision on seen attacks but zero recall on novel threats.

Tier 2 — One-Class SVM

Kernel boundary detector trained on benign-only data. Catches some zero-day threats via anomaly boundary estimation.

Tier 3 — Autoencoder (This Work)

Deep anomaly scorer that learns the normal manifold. Excels at detecting volumetric and timing-based attacks. Threshold tunable without retraining.

Defense-in-Depth: RF catches known patterns, OCSVM provides kernel-based novelty detection, and the Autoencoder supplies continuous anomaly scoring. A future hybrid layer will fuse all three signals for maximum coverage.
