% ---- Abstract ----
\begin{abstract}
This report presents an adaptive intrusion detection pipeline applied to
the CIC-IDS-2018 network-traffic benchmark, with an emphasis on
\emph{zero-day} threat recognition. The processed dataset contains
2{,}097{,}149 labeled flows derived from CIC-IDS-2018 and spans four
traffic classes. Two complementary models are studied: a supervised
\textbf{Random Forest} configured under a seen-versus-unseen attack
split, and a \textbf{One-Class SVM} trained exclusively on benign
traffic. The Random Forest achieved 0.4396 accuracy, 1.00 precision,
0.178 recall, 0.303 F1, and 0.00 zero-day recall; the One-Class SVM
achieved 0.907 precision, 0.145 recall, 0.249 F1, and 0.145 zero-day
recall. The complementary nature of the two models—high confidence on
known threats versus novelty sensitivity for new ones—demonstrates the
necessity of a hybrid evaluation strategy for adaptive cyber-physical
security.
\end{abstract}

\section{Introduction and Problem Statement}
\label{sec:intro}

Cyber-physical systems (CPS) integrate networked computation with
physical processes, making them high-value targets for sophisticated
adversaries. Intrusion detection in such environments must handle two
fundamentally different conditions: (i) classifying attacks that appear
in historical training data, and (ii) flagging novel attack traffic not
present during training—the \emph{zero-day} scenario. Relying on purely
supervised classifiers addresses the first condition but systematically
fails the second \cite{buczak2016}, because attack taxonomies evolve
faster than labeled corpora.

This project addresses both conditions by pairing a discriminative
classifier with an anomaly detector applied to CIC-IDS-2018 network
flow data \cite{sharafaldin2018}. The practical pipeline:

\begin{itemize}[leftmargin=*,noitemsep]
  \item cleans and standardizes raw traffic features;
  \item studies class balance and correlation structure;
  \item reduces redundant and unstable features;
  \item evaluates a supervised baseline for known attacks; and
  \item evaluates an anomaly model for zero-day detection.
\end{itemize}

\section{Literature Review}
\label{sec:lit}

\textbf{CIC-IDS-2018.} Sharafaldin et al.\ introduced CIC-IDS-2018 to
address benchmark limitations such as outdated attack patterns, weak
traffic diversity, and unrealistic feature representations
\cite{sharafaldin2018}. The dataset provides modern network flow
statistics with explicit attack labels, making it well-suited for both
supervised and anomaly-based evaluation.

\textbf{Random Forest.} Breiman's ensemble formulation reduces
prediction variance through bagging and random feature subsampling
\cite{breiman2001}. In intrusion detection, this robustness is
valuable because packet-derived features exhibit collinearity, noise,
and heterogeneous scales. Recent surveys confirm Random Forest remains
a strong tabular baseline for network security tasks \cite{buczak2016}.

\textbf{One-Class SVM.} Schölkopf et al.\ formalized One-Class SVM as
a support estimation problem \cite{scholkopf2001}. This framing fits
zero-day detection naturally: instead of requiring exhaustive malicious
labels, the model estimates the boundary of normal traffic and flags
deviating samples. Subsequent work by Tax and Duin extended this to
general support vector data description.

\textbf{Anomaly Detection.} Chandola et al.\ survey anomaly detection
as a family of techniques built on diverse assumptions about what
``normal'' looks like \cite{chandola2009}. Their taxonomy clarifies why
both supervised and anomaly-based detectors contribute complementary
information—the supervised model excels when representative attack
labels exist, while the anomaly detector hedges against label poverty.

\textbf{Network Security Challenges.} Buczak and Guven highlight
recurring ML challenges in cyber-security: label quality, class
imbalance, concept drift, and feature redundancy \cite{buczak2016}.
Garcia-Teodoro et al.\ further argue that signature-based systems
cannot keep pace with polymorphic or previously unseen attacks, and
advocate for anomaly-based IDS as a complementary layer.

\section{Feature Engineering}
\label{sec:fe}
The feature-engineering pipeline follows four main phases:
\begin{enumerate}[leftmargin=*,noitemsep,topsep=0pt]
  \item \textbf{Identifier removal.} Drop \texttt{Flow~ID}, source/
        destination IPs, port identifiers, and timestamps, which leak
        non-general information.
  \item \textbf{Numeric coercion.} Coerce all non-label columns to
        numeric types; replace infinite values with \texttt{NaN} and
        impute or drop sparse features.
  \item \textbf{Constant removal.} Discard features with zero variance,
        as they carry no discriminative information.
  \item \textbf{Correlation pruning.} Apply a Pearson correlation
        threshold of $\rho = 0.98$: retain domain-preferred features
        and drop redundant partners. This reduces the feature set from
        82 raw columns to 52 meaningful columns.
\end{enumerate}
%
Figures~\ref{fig:fe-hist} and \ref{fig:fe-corr} confirm that the
cleaned feature set has a more compact correlation structure
compared to the raw features, supporting both model interpretability

\section{Theoretical Background}
\label{sec:theory}

\subsection{Random Forest}

A Random Forest trains $B$ decision trees $\{T_b\}_{b=1}^{B}$, each on
a bootstrap sample of the training data with a random subset of
$m \ll p$ features at every split. Classification prediction is:
\begin{equation}
  \hat{y}(\mathbf{x}) = \underset{c}{\arg\max}\;\sum_{b=1}^{B}
  \mathbf{1}\!\left[T_b(\mathbf{x}) = c\right].
  \label{eq:rf}
\end{equation}
Bagging reduces variance without increasing bias; feature subsampling
de-correlates trees and further lowers the ensemble variance
\cite{breiman2001}. For network-flow tabular data with many correlated
numeric predictors, Random Forest provides a strong baseline that does
not require feature scaling.

\subsection{One-Class SVM}

One-Class SVM estimates a decision function $f(\mathbf{x}) = \text{sgn}
\!\left(w^\top\phi(\mathbf{x}) - \rho\right)$ by solving:
\begin{equation}
\min_{w,\,\rho,\,\xi}\;\frac{1}{2}\|w\|^2
  + \frac{1}{\nu n}\sum_{i=1}^{n}\xi_i - \rho
\label{eq:ocsvm-obj}
\end{equation}
subject to
\begin{equation}
w^\top\phi(\mathbf{x}_i) \ge \rho - \xi_i, \qquad \xi_i \ge 0,
\label{eq:ocsvm-cst}
\end{equation}
where $\nu \in (0,1]$ upper-bounds the expected outlier fraction and
$\phi(\cdot)$ is the kernel mapping \cite{scholkopf2001}. An RBF kernel
$k(\mathbf{x},\mathbf{x}') = \exp\!\left(-\gamma\|\mathbf{x}-\mathbf{x}'\|^2\right)$
is used here, which provides implicit infinite-dimensional feature
maps without explicit computation. After standardisation ($\mu=0$,
$\sigma=1$), the model is trained on benign traffic only and evaluated
on both benign and all attack samples.

\subsection{Evaluation Metrics}

Two evaluation perspectives are maintained throughout:

\begin{itemize}[leftmargin=*,noitemsep]
  \item \textbf{Standard supervised metrics:} accuracy, precision,
        recall, and F1-score, measured on the full test set.
  \item \textbf{Zero-day recall:} recall computed \emph{exclusively} on
        the unseen-attack subset, defined as
        \begin{equation}
          \text{ZD-Recall} = \frac{TP_{\text{unseen}}}{TP_{\text{unseen}} + FN_{\text{unseen}}}.
          \label{eq:zdrecall}
        \end{equation}
\end{itemize}

This separation is critical: a model can score high aggregate metrics
while still exhibiting zero zero-day recall—failing the core security
requirement of novelty detection.

\subsection{Interpretation}

The two models serve complementary operational roles:

\begin{itemize}[leftmargin=*,noitemsep]
  \item \textbf{Random Forest} achieves perfect precision on training-
        distribution attacks but \emph{zero} zero-day recall.
  \item \textbf{One-Class SVM} achieves lower overall recall but
        \emph{non-zero} zero-day recall, providing a hedge against
        novel attack families.
\end{itemize}

This confirms the central thesis: a purely supervised classifier is
insufficient when the evaluation target includes unseen attacks. A
hybrid pipeline combining both models is therefore more defensible—the
supervised component provides high-confidence classification for known
threats, while the anomaly detector acts as a novelty sentinel.

\section{Repository and Code Quality}
\label{sec:repo}

The repository mirrors a standard data-science workflow:

\begin{itemize}[leftmargin=*,noitemsep]
  \item \texttt{pipeline/} — preprocessing and cleaning scripts;
  \item \texttt{experiments/eda} — exploratory visualisations;
  \item \texttt{experiments/feature\_engineering} — correlation and
        histogram analysis;
  \item \texttt{experiments/models} — training and evaluation scripts;
  \item \texttt{outputs/plots} — auto-generated figures; and
  \item \texttt{report/} — written deliverables.
\end{itemize}

Recent code-quality improvements include: root-relative path resolution
replacing fragile working-directory assumptions; explicit failure on
missing required inputs; safe removal of the stray \texttt{label} row
during preprocessing; crash-hardening for small benign subsets in the
One-Class SVM script; and cache-path hardening for reproducibility in
restricted environments. The primary remaining engineering gap is a
top-level orchestrated runner with lightweight schema validation tests.

\section{Limitations and Next Steps}
\label{sec:limits}

\begin{itemize}[leftmargin=*,noitemsep]
  \item The checked-in artifact covers a subset of the full
        CIC-IDS-2018 attack space; extended evaluation requires
        regeneration from the complete raw archive.
  \item Feature engineering is correlation-based only; feature
        importance-driven selection and ablation studies remain future work.
  \item Full hyperparameter tuning and repeated cross-validation are
        not yet part of the pipeline.
  \item Concept drift evaluation and deployment-time latency profiling
        are deferred to Phase 2.
  \item The current Random Forest zero-day recall of 0.00 highlights
        the need for semi-supervised or transfer-learning extensions.
\end{itemize}

The next iteration should regenerate the processed artifact from the
full raw data, persist model metrics to disk, and incorporate
detection-threshold analysis for the anomaly component.

% ============================================================
\section{Conclusion}
\label{sec:conclusion}

This project delivers a coherent Phase-1 pipeline for adaptive
cyber-physical intrusion detection on CIC-IDS-2018. It integrates
literature-grounded motivation, dataset quality analysis, structured
feature engineering, rigorous mathematical treatment of both models,
and empirical results evaluated through a dedicated zero-day recall
metric. The key finding is that a supervised classifier alone cannot
satisfy the open-world security objective: the Random Forest captures
known attack patterns with high precision but fails entirely on unseen
attack classes, while the One-Class SVM provides meaningful zero-day
sensitivity at the cost of conservative recall. Combining both
paradigms is the recommended direction for adaptive security systems
operating in dynamic threat environments.
